# 프롬프트 2 — Contradiction Map (모순 지도)

> Co-STORM의 *moderator* + *mind map* 정신을 차용. 다섯 관점이 **어디서 부딪히는지**를
> 명시적으로 도식화한다. 단일 관점이 절대 못 보는 "긴장 지점"이 여기서 드러난다.
> 입력: 프롬프트 1의 출력(다섯 관점 + 출처). `{{SCAN_OUTPUT}}`에 붙여넣기.

---

```
아래는 주제 "{{TOPIC}}"에 대한 다섯 전문가(회의주의자·경제학자·역사학자·학자·미래학자)의
관점과 출처입니다:

{{SCAN_OUTPUT}}

이제 모더레이터로서, 이 다섯 관점이 서로 충돌·긴장하는 지점을 지도로 만드세요.

1. 합의 지점 (Consensus): 둘 이상의 전문가가 같은 결론에 도달한 부분.
   각 합의에 어떤 전문가들이 동의하는지 + 그들의 출처를 묶어 표기.

2. 모순 지점 (Contradiction): 전문가들이 정면으로 충돌하는 부분.
   각 모순에 대해:
   - 무엇에 대한 충돌인가 (한 줄)
   - A 전문가의 입장 + 출처
   - B 전문가의 입장 + 출처
   - 이 충돌이 미해결인 이유 (데이터 부족? 가치관 차이? 시간 지평 차이?)

3. 사각지대 (Blind Spot): 다섯 전문가 누구도 다루지 않았지만 중요한 질문.

4. 가장 생산적인 긴장 (Key Tension): 위 모순 중 이 주제의 핵심을 가르는 단 하나.
   왜 이것이 가장 중요한지.

규칙:
- 모순을 인위적으로 봉합하지 마세요. 진짜 충돌은 충돌로 남겨두는 것이 가치입니다.
- 각 입장에 원래 전문가의 출처를 그대로 끌고 오세요. 출처 없는 모순은 무효.
- "양쪽 다 일리 있다"로 끝내지 말고, 무엇이 해결되면 충돌이 풀리는지 명시.

출력: 합의 / 모순(표) / 사각지대 / 핵심 긴장.
```

---

## 사용 메모

- 이 단계의 산출물(특히 "핵심 긴장")이 다음 종합(프롬프트 3)의 **논지 축**이 된다.
- 모순을 죽이지 않는 것이 STORM이 단일 RAG보다 나은 이유: "아무도 못 본 것"은 대개 충돌 지점에 숨어 있다.
- 본 스킬에서는 5관점 에이전트이 *다른 모델*이라 모순이 더 선명하다 (같은 모델 내 역할극보다 진짜 이견).
