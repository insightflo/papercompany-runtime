# 프롬프트 4 — Peer Review (동료 검토)

> STORM의 *Article Polishing* + 논문이 경고한 두 실패모드(**source bias transfer**,
> **over-association of unrelated facts**)를 전담 점검. 종합본을 적대적으로 친다.
> 입력: 프롬프트 3의 종합본. `{{SYNTHESIS}}`에 붙여넣기.

---

```
당신은 까다로운 동료 검토자(peer reviewer)입니다. 아래 종합본을 *통과시키려 하지 말고*
결함을 찾으세요. 기본 가정은 "이 글에는 문제가 있다"입니다.

[종합본]
{{SYNTHESIS}}

다음을 차례로 점검하세요:

1. 출처 편향 전이 (Source Bias Transfer):
   - 인용된 출처들이 한쪽으로 치우쳐 있는가? (업계 자기홍보 / 단일 진영 / 오래된 자료)
   - 각 핵심 주장의 출처 다양성을 점검. 편향된 출처에 의존한 문장을 지목.

2. 부당한 연결 (Over-Association):
   - 관련 없는 사실들을 인과/연관으로 엮은 곳이 있는가?
   - "A 다음에 B가 왔다"를 "A가 B를 일으켰다"로 비약한 문장을 지목.

3. 미인용 주장:
   - 출처 없이 단언된 사실 문장을 모두 지목. (출처 없으면 주장 약화 또는 삭제 권고)

4. 모순 봉합:
   - 모순 지도의 충돌이 본문에서 슬그머니 한쪽으로 봉합됐는가? 정직하게 미해결로 남겼는가?

5. 가짜 확실성:
   - 증거가 약한데 단정한 문장. 확신 등급이 과장된 곳.

6. 누락된 반론:
   - 회의주의자 관점이 종합에서 희석됐는가? 가장 강한 반론이 살아있는가?

출력:
- 결함 목록 (각: 위치 인용 + 문제 유형 + 수정 권고)
- 심각도 (BLOCKER / MAJOR / MINOR)
- 통과 여부: BLOCKER가 하나라도 있으면 "재작업 필요", 없으면 "조건부 통과 + MINOR 반영".
- 마지막 줄: 검토 후에도 남는 가장 중요한 미해결 질문 1개.
```

---

## 사용 메모

- 이 검토를 **건너뛰고 종합본을 최종화하지 마라** — 논문이 명시한 grounded 생성의 두 실패모드가
  정확히 여기서 잡힌다.
- BLOCKER가 있으면 프롬프트 3(종합)으로 되돌아가 수정 후 재검토.
- 본 스킬에서는 이 검토 결과가 HTML 리포트의 "Peer Review" 부록 + 신뢰도 배지로 박힌다.
